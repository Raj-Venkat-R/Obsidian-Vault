## üîπ Problem Summary
You are given an array where each element represents the stock price on a given day.  
You can complete **as many transactions as you want** (buy one stock and sell it later), but you must sell the stock before buying again.  
The goal is to calculate the **maximum profit** possible.

---

## üîπ My First Thought (Initial Approach)
At first, it felt like I needed to explicitly decide on buy and sell days.  
But since multiple transactions are allowed, I realized I only need to take profit whenever the price increases.

---

## üîπ Key Observation
- Multiple transactions are allowed
- Profit can be accumulated from **every increase**
- It is not necessary to track exact buy and sell days
- Small profits added together give the maximum profit

---

## üîπ Final Approach (Step-by-Step in English)
1. Traverse the price array from the second day onward
2. Compare the current day‚Äôs price with the previous day‚Äôs price
3. If today‚Äôs price is higher than yesterday‚Äôs:
   - Add the difference to the total profit
4. If today‚Äôs price is lower or equal:
   - Do nothing
5. Continue until the end of the array

## üîπ Java Code (Final)
> ‚ö†Ô∏è Code added after fully understanding the logic.

```java
class Solution {
    public int maxProfit(int[] prices) {
        int profit = 0;
        for (int i = 1; i < prices.length; i++) {
            if (prices[i] > prices[i - 1]) {
                profit += prices[i] - prices[i - 1];
            }
        }
        return profit;
    }
}
```

## üîπ Time & Space Complexity
- **Time Complexity:** O(n)  
  The price array is traversed once, comparing adjacent days.
- **Space Complexity:** O(1)  
  Only a single variable is used to store the total profit.
