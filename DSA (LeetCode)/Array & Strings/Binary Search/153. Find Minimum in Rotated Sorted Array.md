Given a rotated sorted array with **no duplicates**, find the **minimum element** in O(log n) time.

The array was originally sorted in ascending order and then rotated.

## ğŸ”¹ Key Observation
In a rotated sorted array:
- One half is always sorted
- The **minimum lies in the unsorted half**
- Comparing `nums[mid]` with `nums[high]` tells where the minimum is

---

## ğŸ”¹ Final Approach (Step-by-Step in English)

1. Use two pointers `low` and `high`.
2. While `low < high`:
   - Calculate `mid`.
   - If `nums[mid] > nums[high]`, the minimum is in the **right half**.
   - Otherwise, the minimum is in the **left half including mid**.
3. When loop ends, `low` points to the minimum element.

## ğŸ”¹ Java Code (Final)
> âš ï¸ Code added after fully understanding the logic.

```java
class Solution {
    public int findMin(int[] nums) {
        int low = 0;
        int high = nums.length - 1;

        while (low < high) {
            int mid = (low + high) / 2;

            if (nums[mid] > nums[high]) {
                // minimum is in the right half
                low = mid + 1;
            } else {
                // minimum is in the left half (including mid)
                high = mid;
            }
        }
        return nums[low];
    }
}
```
