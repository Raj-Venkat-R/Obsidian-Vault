# Fixed vs Dynamic Arrays

Arrays can be classified based on **size flexibility**:
- **Fixed Arrays**
- **Dynamic Arrays**

The core difference is **whether the size can change after creation**.

---

## ğŸ”¹ Fixed Array

### What is a Fixed Array?

A **fixed array** is an array whose **size is decided at creation time and cannot be changed**.

### Java Example
```java
int[] arr = new int[5];
```

- Physical size = 5
- Logical size â‰¤ 5
- Cannot grow or shrink

## ğŸ”¹ Memory View (Fixed Array)

```

```

- Memory block is fixed
- Extra slots remain unused
- Programmer must track logical size

---

## ğŸ”¹ Characteristics of Fixed Arrays

- Fixed physical size
- Stored in contiguous memory
- Very fast index access
- Manual insertion & deletion
- Risk of overflow or wasted memory

---

## ğŸ”¹ Limitations of Fixed Arrays

- Size must be known beforehand
- Cannot handle growing input easily
- Manual resizing is required
- Shifting needed for insert/delete

---

## ğŸ”¹ Dynamic Array

### What is a Dynamic Array?

A **dynamic array** is an array that **automatically grows or shrinks** as elements are added or removed.

> Dynamic arrays internally still use **fixed arrays**.

## ğŸ”¹ Java Dynamic Array: `ArrayList`

In Java, **`ArrayList`** is the most common dynamic array.

```java

```

You do **not** manage:
- Capacity
- Resizing
- Shifting

Java does it for you.

---

## ğŸ”¹ How `ArrayList` Works Internally (Concept)

When the internal array becomes full:
1. A new **larger array** is created
2. All elements are copied
3. Reference points to new array
4. Old array is discarded

```

New â†’ | 10 | 20 | 30 | _ | _ | _ |
```

Resizing is costly but happens **rarely**, so average performance is good.
