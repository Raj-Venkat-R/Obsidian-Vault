## ðŸ”¹ What is Brute Force?

**Brute force** means:
> Trying **all possible ways** to solve a problem  
> without worrying about optimization initially.

In arrays, this usually means:
- Nested loops
- Checking every element
- Checking every pair or subarray

- Two pointers
- Sliding window
- Hashing

- Confusion
- Wrong logic
- Memorized solutions âŒ

Correct flow:
```
Understand problem â†’ Brute force â†’ Optimize
```

If brute force is wrong, optimization will also be wrong.

## ðŸ”¹ How to Think Brute Force (Step-by-Step)

When given an array problem, ask:

1. What is required?
2. Which elements are involved?
3. Do I need:
   - One element?
   - Two elements?
   - A range of elements?
4. Can I check all possibilities?

## ðŸ”¹ Common Brute Force Patterns in Arrays

### 1. Single Loop (Check Every Element)

Used when:
- Finding max / min
- Counting
- Checking condition

Mental model:
```
for each element â†’ do something
```

---

### 2. Double Loop (Check Every Pair)

Used when:
- Comparing two elements
- Finding pairs
- Checking relationships

Mental model:
```
for each element
    for each other element
```

---

### 3. Triple Loop (Check Every Subarray)

Used when:
- Working with ranges
- Calculating sums of subarrays
- Checking all combinations

Mental model:
```
start index
    end index
        process elements
```

## ðŸ”¹ How Optimization Comes from Brute Force

Optimization means:
> Removing **repeated work**

Common repeated work in arrays:
- Recalculating sums
- Rechecking same elements
- Scanning same range again

This leads to:
- Prefix sum
- Sliding window
- Hashing

All optimizations are **brute force + smart idea**.

## ðŸ”¹ When to STOP Brute Force

You stop brute force when:

- Input size is large
- You see repeated computation

But never skip brute force **thinking**.
